Home Page (home.html)

This is the layout for the Home Page, the site's landing page. It has an auto-rotating hero slider, four mission-focused summary cards, and an article directory at the bottom that links out to the site's content pages.

HTML Structure

Hero Slider

The hero is a track of slide divs, and each one carries its photo as a CSS background instead of an img tag. I needed it as a background because the slide animation moves things with transform, and that's easier to do with backgrounds than swapping image sources.

html
<div class="hero-slider-wrapper">
    <div class="hero-slider-track">
        <div class="hero-slide" style="background-image: url(images/hero-bg-img1.jpg);" role="img"
            aria-label="https://pixabay.com/photos/people-industry-construction-3286036/"></div>
        ...
    </div>
</div>

Since a CSS background image has no alt text of its own, I added role="img" and aria-label to each slide so screen readers still get something meaningful instead of nothing. I also updated the slider so it loops properly now: the script clones the first slide and sticks it on the end of the track, so once the slideshow reaches that clone it can snap back to slide 0 with the transition turned off for a moment. That way it loops endlessly instead of jumping back visibly.

Mission & Goal Cards

Four cards summarise the site's main themes. Each one has a credited photo, a short write-up, and a small list of outcomes or actions underneath.

html
<div class="card">
    <h2>The Crisis of Legacy Infrastructure</h2>
    <div class="mgp-card-content-container">
        <div class="mgp-card-img">
            <figure>
                <img src="images/mgp-card-img1.jpg" alt="Explore Dubai's intricate roadways...">
                <figcaption><a class="generic-link" href="...">Photo by Kate Trysh - Pexels</a></figcaption>
            </figure>
        </div>
        <div class="mgp-description">
            <p>Modern societies are heavily reliant on systems engineered in the previous century...</p>
        </div>
    </div>
    <div class="mgp-card-highlights">
        <h3>Key Outcomes :</h3>
        <ul>
            <li>40% reduction in regional energy transmission losses.</li>
            ...
        </ul>
    </div>
    <div class="mgp-card-bottom-text">Local communities and civic planners.</div>
</div>

I split the image and the description text into their own wrapper divs (.mgp-card-img and .mgp-description) instead of mixing them together, so I could size and wrap each one on its own. I also moved the highlights block outside the content container rather than nesting it inside, which keeps every card's outcomes list sitting at the same depth no matter how long the description text ends up being. Figure and figcaption are used again here to keep each photo tied to its credit, same as on the content pages.

Article Linking Section

A "Read our articles..." section closes the page off. There's one big featured thumbnail followed by a grid of smaller ones, and they all link out to the article pages.

html
<div class="mini-article-thumbnail clickable-box" data-href="content_ST1.html" role="link" tabindex="0"
    id="st2_article">
    <div class="mini-article-img-container"><img src="images/gallery/card3_img.jpg"></div>
    <div class="article-meta-data">
        <span>Thimithri vidanapathirana</span>
        <span>03 May 2026</span>
    </div>
    <div class="article-content">
        <h2>Integrating Micro-Enterprises Into Modern Financial Value Chains</h2>
        <p>Discover how grassroots businesses bypass traditional banking barriers...</p>
    </div>
    <div class="article-read-more"><span>Read more</span><svg>...</svg></div>
</div>
js

document.addEventListener('click', (event) => {
    const clickableElement = event.target.closest('.clickable-box');

    if (clickableElement) {
        console.log(`Article click event - id: ${clickableElement.id}`);
        window.location.href = clickableElement.dataset.href;
    }
});

Instead of adding a click listener to every single thumbnail, there's one listener on the whole document that checks if whatever was clicked is inside a .clickable-box, then reads its data-href and navigates there. That means new thumbnails just need the class and the attribute, nothing extra in the script. Each thumbnail also has role="link" and tabindex="0" so it's still discoverable and focusable even though it's a div and not a real anchor tag.

CSS Architecture

Hero Slider Track

I used Grid instead of Flexbox for the slider track because each slide needs to take up exactly one full-width column that can be shifted together as a unit.

css
.hero-slider-track {
    display: grid;
    grid-auto-flow: column;
    grid-auto-columns: 100%;
    width: 100%;
    height: 100%;
}

Why Grid? grid-auto-flow: column with grid-auto-columns: 100% lays out as many equal columns as there are slides automatically. Adding a seventh slide, or the cloned one for the loop, doesn't need any CSS changes at all.

Layered Hero Overlays

The gradient trapezium and the frosted glass pane sit on top of the slider using absolute positioning and a shared clip-path.

css
.hero-overlay-1 {
    position: absolute;
    width: 60%;
    height: 100%;
    z-index: 2;
    clip-path: polygon(0% 0%, 75% 0%, 100% 100%, 0% 100%);
    background-image: linear-gradient(to right top, #fd6925, ..., #5e256d);
}
.hero-overlay-2 {
    position: absolute;
    width: 60%;
    height: 100%;
    z-index: 3;
    clip-path: polygon(0% 0%, 75% 0%, 100% 100%, 0% 100%);
    background-color: rgba(255, 255, 255, 0.6);
    backdrop-filter: blur(40px);
}

Both layers use the same clip-path so they line up, and that's what gives the hero its angled, sliced look instead of a plain rectangle. The backdrop-filter: blur(40px) lets the gradient and the photo behind it show through softly while keeping the heading text easy to read.

Mission Card Layout

Cards use Flexbox in column direction, with a small trick to keep the bottom text strip pinned in place no matter how much content is above it.

css
.card {
    display: flex;
    flex-direction: column;
    gap: 10px;
}
.mgp-card-bottom-text {
    margin-top: auto;
    border-top: 1px solid #FD6925;
}

margin-top: auto inside a flex column pushes the bottom text all the way down to the bottom of the card, so every card's footer strip lines up even if the description text is a different length.

Some design/styling choices

The hero gradient (orange fading into purple) is the boldest colour moment on the page, and everywhere else stays fairly neutral so the gradient panel is what draws the eye first. Article thumbnails get a light background hover state and their "Read more" arrow nudges forward on hover, which matches the hover style used on links elsewhere on the site. Padding and gaps are kept consistent across the mission cards and the article grid so they feel like part of the same layout rather than two separate sections stuck together.

Core Rationale

Keeping the hero's slideshow, gradient and glass pane as separate absolutely-positioned layers means I can tweak any one of them without affecting the others. The role and aria-label attributes on the slides, and the role/tabindex on the article thumbnails, bring back the accessibility meaning that plain divs would otherwise lose. Reusing the same card, figure and figcaption patterns, and the same hover feedback style, from the rest of the site keeps the home page feeling consistent with the content pages instead of like a separate design.

Accessibility

Every hero slide and mission card image has descriptive alt text or an equivalent aria-label. The decorative watermark text in the background is marked aria-hidden so it doesn't get read out loud. Article thumbnails can be reached by keyboard thanks to tabindex, and they're announced as links, though right now the click handling only listens for mouse clicks. Adding a keydown handler for Enter and Space would be a good follow-up so the thumbnails can actually be activated by keyboard too, not just focused.

Gallery Page (gallery.html)

Technical Description

The gallery is a responsive grid of thumbnail cards built with CSS Grid, so the number of columns adjusts automatically to the viewport width instead of relying on fixed breakpoints. Clicking a thumbnail opens a JavaScript-driven modal that displays a larger version of the image along with its caption and credit, without navigating away from the gallery grid underneath. An accessibility control panel sits above the gallery, toggling classes on the <body> element which the CSS then reads to adjust animation and sizing rules.

Accessibility

Every thumbnail and modal image has a full-sentence alt description. The modal can be opened and closed with the keyboard (Enter/Space to open, Escape to close), and focus is moved into the modal when it opens and returned to the triggering thumbnail when it closes. The accessibility control panel is built from real form controls (<button>/<input type="checkbox">) with visible labels rather than icon-only controls.

Content Page (content_ST1.html)

This is the layout for the Content Page (used for the "Integrating Micro-Enterprises" article). It's a shared template (.content-page), so these styles apply site-wide.

Design Inspiration: Based on Odama's work on Dribbble. It uses a full-bleed hero banner with the title layered right on top of the photo instead of a boring text header.

HTML Structure

Hero Banner

Starts with a full-width banner containing the background image, credits, an editor's note link, and the heading overlay.

html
<div class="content-page-banner">
    <img src="images/content-page-banner-img-w2212570.jpg" class="content-page-banner-image"
        alt="Two elderly Vietnamese women weaving traditional bamboo fish traps...">
    <div class="content-page-banner-img-reference">...</div>
    <a class="generic-link content-page-banner-editor-link" href="pageEditor_ST1.html">Editor's page</a>
    <div class="content-page-banner-heading">
        <h1>Integrating Micro-Enterprises Into Modern Financial Value Chains</h1>
        <p>By: Eshan Amarakoon</p>
    </div>
</div>
  • Accessibility: Every image uses a full-sentence alt tag (no filenames or blank strings) so screen readers get the full context.
  • SEO: The <h1> is only used once for the main title to keep the document hierarchy clean.
  • Links: Used actual <a> tags for credits and links so they are keyboard accessible.

Two Column Layout

The main body splits into a sticky sidebar navigation and the actual article content.

html
<div class="article-content-container">
    <aside class="article-content-sidebar">
        <nav>
            <ul>
                <li><a href="#microfinance">Supporting Microfinance...</a></li>
                ...
            </ul>
        </nav>
    </aside>
    <div class="article-content">
        <section class="content-page-section-card" id="microfinance">
            <h2>Supporting Microfinance and Community-Led Credit</h2>
            ...
        </section>
        ...
    </div>
</div>
  • <aside> and <nav> Marks the sidebar as supplementary and tells search engines/screen readers it's an in-page table of contents.
  • Each <section> has a unique id matching the sidebar links for smooth-scroll navigation.
  • <h2> tags Keeps sections neatly organized under the main <h1>.

Other content Elements

A few specific tags were repurposed to handle tables, lists, and images cleanly.

HTML
<!-- For tabular comparisons -->
<table>
    <caption><h3>Structural Barriers vs. Grassroots Financial Solutions</h3></caption>
    <thead>
        <tr><th>Traditional Banking Obstacles</th><th>Grassroots & Microfinance Solutions</th></tr>
    </thead>
    <tbody>
        <tr><td>High collateral requirements...</td><td>Trust-based or group-guaranteed lending...</td></tr>
    </tbody>
</table>

<!-- For boxed action plans -->
<fieldset>
    <legend><h3>The Merchant Digital Onboarding Roadmap</h3></legend>
    <ol>
        <li><strong>Inventory Assessment:</strong> Document and photograph...</li>
    </ol>
</fieldset>

<!-- For image captions -->
<figure class="article-content-fig">
    <img src="images/article-content-img1-w2212570.jpg" alt="A factory worker...">
    <figcaption>Photo by <a href="...">Sam Moghadam</a> on <a href="...">Unsplash</a></figcaption>
</figure>
  • <table> Used strictly for actual data comparisons. Includes <th> tags for accessibility.
  • <fieldset> and <legend>: Borrowed these form tags because they give us a nice, built-in bordered box with an inline title.
  • <figure>: Tethers the image and its source credit together as a single unit.

Interactive Poll

Design Inspiration: The embedded poll was inspired by the reader polls IGN runs at the bottom of their articles — a quick, opinion-style question with a live vote count that reveals each option as a percentage bar once you've voted.

html
<section class="poll">
    <div class="poll-container">
        <div class="poll-header">
            <span id="poll-live-btn">live</span>
            <h2>When purchasing daily essentials or services...</h2>
        </div>
        <div class="poll-body" id="poll-body-form">
            <form id="dummy-article-poll" action="..." method="get">
                <ul>
                    <li>
                        <input type="radio" id="convenience" name="corporate-over-local"
                            value="convenience" required>
                        <label for="convenience">Convenience and speed...</label>
                    </li>
                    ...
                </ul>
            </form>
        </div>
        <div class="poll-body" id="poll-body-result" style="display: none;"></div>
        <div class="poll-footer">...</div>
    </div>
</section>

A small script listens for the form's submit event, tallies the vote against a dummy data object, then swaps #poll-body-form for #poll-body-result and renders each option as a percentage bar — the same reveal-on-vote pattern IGN uses, just built from scratch with vanilla JS instead of a third-party poll widget.

  • Accessibility: Options use real <input type="radio"> / <label for="..."> pairs so the poll is fully keyboard and screen-reader operable, not just clickable <div>s.
  • Progressive enhancement: The <form> still has a real action/method, so it degrades gracefully if JavaScript fails to load.
  • Feedback: Submitting the form disables the button and swaps its label to "Vote Submitted" so users get clear confirmation their vote registered.

CSS Architecture

Article layout

Used Grid for the main container because I needed a fixed width center column flanked by flexible side gutters.

css
.article-content-container {
    display: grid;
    grid-template-columns: minmax(290px, 1fr) 900px 1fr;
    gap: 40px;
    margin: 0 auto;
    margin-top: 50px;
}

Why Grid? It lets us pin the middle reading column exactly to 900px in one line of code. Flexbox would require too much guessing with flex-grow and flex-basis.

Hero Banner Overlay

To get the text floating over the image, the container is relative and the text block is absolute.

css
.content-page-banner {
    height: 60vh;
    position: relative;
}
.content-page-banner-heading {
    position: absolute;
    top: 50%;
    left: 5%;
    z-index: 2;
    background-color: rgba(255, 255, 255, 0.1);
    backdrop-filter: blur(10px);
}

backdrop-filter: blur() with background-color: rgba(255, 255, 255, 0.1); keeps the text perfectly legible over any background image without completely hiding the photo behind a solid box.

Sticky Sidebar

The table of contents stays pinned to the screen while the user reads.

css
.article-content-sidebar {
    position: sticky;
    top: 20px;
    height: fit-content;
}

Using position: sticky gives me a persistent sidebar natively and no JavaScript scroll listeners required.

Some design/styling choices

  • Colors are mostly neutral tones (#F4F7F9 background, #475569 text). It's easy and comfortable for eye. And not too fancy
  • All links and buttons have obvious :hover and :active states (like a slight translateX(4px) nudge) so users know they are clickable.
  • Consistent grid gaps (40px) and padding keep the card layout looking unified.

Core Rationale

Used tags like <aside>, <nav>, and <section> instead of <div>. It drastically improves SEO and screen-reader accessibility. The sticky sidebar makes navigating a long article effortless. Frosted glass ensures readability without breaking the visual aesthetic. Descriptive alt text keeps the page screen-reader friendly (no blank names or raw filenames). Using <figcaption> anchors photo credits directly to the image for everyone. Reusing the .content-page-section-card and .card classes everywhere keeps the look unified so it feels like a single, solid design system.

Accessibility

Beyond the semantic tags above: every image (banner, article figures) carries a full-sentence alt description; the heading hierarchy uses exactly one <h1> per page with <h2> for each article sub-section; and every credit/citation link is a real, keyboard-focusable <a> tag rather than plain text.
