Content Page validation report

This one had the most errors out of my three pages. My first figure had a completely broken img tag because I'd somehow put an extra pair of quote marks around the file path, so the validator couldn't even read it as a proper attribute. There was also a stray "<" character just sitting on its own line in the banner caption, which isn't allowed on its own outside of a tag. On top of that all four of my images were still pointing at the absolute file path on my own laptop (C:\Users\User\Desktop\...) instead of a relative path, which the validator flags since backslashes aren't valid in a URL. I fixed the broken tag, removed the stray character, and switched every image over to a relative images/contentimages/ path instead.
